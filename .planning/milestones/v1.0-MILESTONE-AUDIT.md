## Integration Findings

### INT-01: _stop_event global missing (LOW)

**File:** `orchestrator/app.py` line 19
**Issue:** `_stop_event` assigned in `startup()` without `global _stop_event` declaration. The module-level `_stop_event = None` is never updated. `shutdown()` checks `if _stop_event:` which evaluates to `None` (falsy), so `.set()` is never called.
**Impact:** Consumers stop via Quart's `CancelledError` instead of the intended `stop_event.set()` path. Slightly noisier shutdown but no data loss or consistency violation. Benchmark and kill-test both pass.
**Fix:** Change `global db` to `global db, _stop_event` in `startup()`.
**Affects:** EVENT-02

### INT-02: Architecture diagram port inaccuracies (INFORMATIONAL)

**File:** `docs/architecture.md` lines 22, 24
**Issue:** Mermaid diagram shows Order Service as `:50053 gRPC` (Order is a gRPC client only, no server) and Payment Service as `:50052 gRPC` (actual port is `:50051` — 50052 only used in test fixtures).
**Fix:** Update diagram labels.
**Affects:** DOCS-02
