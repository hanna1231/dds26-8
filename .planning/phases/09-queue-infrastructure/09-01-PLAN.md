- Call `queue_db.xgroup_create(REPLY_STREAM, REPLY_GROUP, id="0", mkstream=True)`

Function `reply_listener(queue_db, stop_event: asyncio.Event)`:
- Async function that runs as a Quart background task
- Loop while `not stop_event.is_set()`
- Call `queue_db.xreadgroup(groupname=REPLY_GROUP, consumername=REPLY_CONSUMER, streams={REPLY_STREAM: ">"}, count=BATCH_SIZE, block=POLL_INTERVAL_MS)`
- For each message: decode `b"correlation_id"` and `b"result"` fields (bytes -> str)
- Parse result JSON via `msgspec.json.decode(result_str.encode())`
- Look up `pending_replies.get(correlation_id)` -- if Future exists and not done, call `future.set_result(result_dict)`
- XACK the message after processing
- Wrap xreadgroup in try/except: CancelledError re-raise, other exceptions log warning + sleep(1) (same pattern as consumers.py)
- Log info on start and stop

Important: Use `b"correlation_id"` and `b"result"` for field access since `decode_responses=False`.

Core function `send_command(stream: str, command: str, payload: dict, timeout: float = COMMAND_TIMEOUT) -> dict`:
- Generate `correlation_id = str(uuid.uuid4())`
- Create `future = asyncio.get_event_loop().create_future()`
- Store in `pending_replies[correlation_id] = future`
- XADD to stream with fields: `{"correlation_id": correlation_id, "command": command, "payload": msgspec.json.encode(payload).decode()}`
- Use `maxlen=STREAM_MAXLEN, approximate=True` on XADD
- `try: result = await asyncio.wait_for(future, timeout=timeout)` then `return result`
- `except asyncio.TimeoutError: return {"success": False, "error_message": "queue timeout"}`
- `finally: pending_replies.pop(correlation_id, None)`
