**orchestrator/app.py** — Quart HTTP shell following stock/app.py pattern:

```python
import os
import redis.asyncio as redis
from quart import Quart, jsonify
from grpc_server import serve_grpc, stop_grpc_server
from client import init_grpc_clients, close_grpc_clients

app = Quart("orchestrator-service")
db: redis.Redis = None

@app.before_serving
async def startup():
    global db
    db = redis.Redis(
        host=os.environ['REDIS_HOST'],
        port=int(os.environ['REDIS_PORT']),
        password=os.environ['REDIS_PASSWORD'],
        db=int(os.environ['REDIS_DB']),
    )
    await init_grpc_clients()
    app.add_background_task(serve_grpc, db)

@app.after_serving
async def shutdown():
    await stop_grpc_server()
    await close_grpc_clients()
    await db.aclose()

@app.get('/health')
async def health():
    return jsonify({"status": "ok"})

if __name__ == '__main__':
    app.run(host="0.0.0.0", port=5000, debug=True)
```

**orchestrator/grpc_server.py** — gRPC servicer + full SAGA execution:

1. **OrchestratorServiceServicer** class with `StartCheckout` method that calls `run_checkout()`.

2. **`run_checkout(db, order_id, user_id, items, total_cost) -> dict`** (main SAGA execution):

   **Exactly-once check (SAGA-06):**
   - Call `get_saga(db, order_id)`. If exists:
     - state == COMPLETED: return `{"success": True, "error_message": ""}`
     - state == FAILED: return `{"success": False, "error_message": existing["error_message"]}`
     - any other state (in-progress): return `{"success": False, "error_message": "checkout already in progress"}`
   - Call `create_saga_record(db, order_id, user_id, items, total_cost)`. If returns False (race — another process created it), re-read and return existing result.

   **Forward execution:**
   - Parse `items` from `items_json` field (JSON list of {"item_id", "quantity"}).
   - **Reserve stock per item:** For each item, call `reserve_stock(item_id, quantity, f"saga:{order_id}:step:reserve:{item_id}")`. If any fails, transition to COMPENSATING, set error_message, run compensation, return failure.
   - After ALL stock reservations succeed: `transition_state(db, saga_key, "STARTED", "STOCK_RESERVED", "stock_reserved", "1")`.
   - **Charge payment:** Call `charge_payment(user_id, total_cost, f"saga:{order_id}:step:charge")`. If fails, transition to COMPENSATING, set error_message, run compensation, return failure.
   - After payment: `transition_state(db, saga_key, "STOCK_RESERVED", "PAYMENT_CHARGED", "payment_charged", "1")`.
   - **Complete:** `transition_state(db, saga_key, "PAYMENT_CHARGED", "COMPLETED")`. Return success.

3. **`run_compensation(db, saga) -> None`** (SAGA-04, reverse order):

   Per locked user decision: compensation runs in reverse, only undoes completed steps, uses per-step boolean flags.

   - Re-read SAGA hash to get current flags (Pitfall 2 avoidance).
   - **Step 1: Refund payment** — only if `payment_charged == "1"` and `refund_done != "1"`:
     - `retry_forever(lambda: refund_payment(user_id, total_cost, f"saga:{order_id}:step:refund"))`
     - Set `refund_done = "1"` in Redis hash.
   - **Step 2: Restore stock per item** — only if `stock_reserved == "1"` and `stock_restored != "1"`:
     - For each item in `items_json`: `retry_forever(lambda: release_stock(item_id, quantity, f"saga:{order_id}:step:release:{item_id}"))`
     - Set `stock_restored = "1"` in Redis hash.
   - **Finalize:** `transition_state(db, saga_key, "COMPENSATING", "FAILED")`.

4. **`retry_forever(fn, base=0.5, cap=30.0) -> dict`** (SAGA-05):

   ```python
   async def retry_forever(fn, base=0.5, cap=30.0):
       attempt = 0
       while True:
           try:
               result = await fn()
               if result.get("success"):
                   return result
           except Exception as exc:
               logging.warning("compensation retry attempt %d failed: %s", attempt, exc)
           delay = min(cap, base * (2 ** attempt))
           await asyncio.sleep(delay)
           attempt += 1
   ```

5. **`serve_grpc(db)` and `stop_grpc_server()`** — following stock/grpc_server.py pattern exactly, but on port **50053** (Stock=50051, Payment=50052 per Phase 2 decision, Orchestrator=50053 per Pitfall 6).

**Key implementation details:**
- All `import` statements use absolute imports (consistent with Phase 2).
- Business errors from Stock/Payment are in `result["success"]` / `result["error_message"]` — never catch gRPC status codes (Phase 2 decision).
- The `items` parameter in `run_checkout` comes from the proto's repeated LineItem — convert to list of dicts: `[{"item_id": item.item_id, "quantity": item.quantity} for item in request.items]`.
- Use `import json` for parsing `items_json` from SAGA record (stored as JSON string by saga.py).
- Use `import logging` and `import asyncio` (stdlib only — no tenacity).
