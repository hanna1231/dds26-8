Create `tests/test_saga.py` with the following tests:

**Test 1: `test_saga_record_created_before_side_effects`** (SAGA-01)
- Call `create_saga_record()` directly with test data.
- Verify Redis hash exists with state=STARTED and all expected fields.
- Verify `stock_reserved`, `payment_charged`, `refund_done`, `stock_restored` all "0".

**Test 2: `test_saga_state_transitions_valid`** (SAGA-02)
- Create a SAGA record.
- Call `transition_state()` through the valid path: STARTED -> STOCK_RESERVED -> PAYMENT_CHARGED -> COMPLETED.
- Verify each transition returns True.
- Verify final state in Redis is COMPLETED.

**Test 3: `test_saga_state_transition_invalid_rejected`** (SAGA-02)
- Create a SAGA record (state=STARTED).
- Attempt invalid transition: STARTED -> COMPLETED (should return False).
- Attempt invalid transition: STARTED -> FAILED (should return False).
- Verify state is still STARTED.

**Test 4: `test_checkout_happy_path`** (SAGA-03)
- Seed Stock Redis with an item (sufficient stock + price).
- Seed Payment Redis with a user (sufficient credit).
- Call `StartCheckout` via orchestrator gRPC stub with valid order_id, user_id, items, total_cost.
- Verify response: `success=True`, `error_message=""`.
- Verify SAGA record state is COMPLETED in orchestrator Redis.
- Verify Stock item stock was decremented.
- Verify Payment user credit was decremented.

**Test 5: `test_checkout_insufficient_stock_compensates`** (SAGA-04)
- Seed Stock Redis with an item (stock=0).
- Seed Payment Redis with a user (sufficient credit).
- Call StartCheckout.
- Verify response: `success=False`, error_message contains "insufficient stock".
- Verify SAGA record state is FAILED.
- Verify payment was NOT charged (credit unchanged).

**Test 6: `test_checkout_insufficient_credit_compensates`** (SAGA-04)
- Seed Stock Redis with an item (sufficient stock).
- Seed Payment Redis with a user (credit=0).
- Call StartCheckout.
- Verify response: `success=False`, error_message contains "insufficient credit".
- Verify SAGA record state is FAILED.
- Verify stock was restored (stock back to original value).

**Test 7: `test_checkout_duplicate_returns_original`** (SAGA-06)
- Complete a successful checkout (test_checkout_happy_path setup).
- Call StartCheckout again with the same order_id.
- Verify response: `success=True` (returns original result, not re-executed).
- Verify stock and credit unchanged from first checkout (no double charge).

**Test 8: `test_saga_duplicate_creation_prevented`** (SAGA-01 atomicity)
- Call `create_saga_record()` for an order_id — returns True.
- Call `create_saga_record()` again for the same order_id — returns False.

**Test 9: `test_compensation_retries_until_success`** (SAGA-05)
- Seed Stock Redis with an item (sufficient stock) and Payment Redis with a user (sufficient credit).
- Monkey-patch (or mock) the orchestrator's `release_stock` gRPC call to raise `grpc.aio.AioRpcError` (status UNAVAILABLE) on the first 2 invocations, then succeed on the 3rd.
- Call StartCheckout with a `total_cost` exceeding user credit so payment fails and compensation triggers stock restore.
- Verify the patched `release_stock` was called exactly 3 times (2 failures + 1 success).
- Verify the SAGA record reaches FAILED state (compensation completed, not silently dropped).
- Verify stock quantity is restored to its original value after the successful retry.
- This proves `retry_forever()` keeps retrying with backoff until compensation succeeds, satisfying SAGA-05.

**Test 10: `test_idempotency_keys_prevent_duplicate_side_effects`** (IDMP-01, IDMP-02, IDMP-03)
- Seed Stock Redis with an item (stock=10) and Payment Redis with a user (sufficient credit).
- Call StartCheckout with a valid order — succeeds, stock decremented and payment charged.
- Record stock quantity and user credit after the first checkout.
- Manually reset the SAGA record state in orchestrator Redis back to STARTED (simulating a re-execution).
- Call `run_checkout()` directly (or replay the SAGA steps) with the same order_id.
- Verify stock quantity is unchanged from after first checkout (Lua idempotency rejected duplicate subtract).
- Verify user credit is unchanged from after first checkout (Lua idempotency rejected duplicate pay).
- This proves Phase 2 Lua-based idempotency (IDMP-01 for stock, IDMP-02 for payment) correctly rejects replayed operations using SAGA-generated idempotency keys. IDMP-03 (concurrency/overselling protection) is implicitly covered since the Lua script is atomic.

**Implementation notes:**
- Use `pytest.mark.asyncio` on all tests.
- Use the `clean_orchestrator_db` fixture for each test.
- Seed Stock and Payment test data using their Redis connections from conftest (direct `db.set()` with msgpack-encoded StockValue/UserValue).
- Import `StockValue` from `stock.grpc_server` and `UserValue` from `payment.grpc_server` for seeding.
- Use unique order_ids per test (uuid4) to avoid cross-test interference even without cleanup.
- For the gRPC call tests, use the `orchestrator_stub` fixture.
- For the direct saga.py tests, use the `orchestrator_db` fixture.
