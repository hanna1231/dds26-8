  <action>
    Add three functions and their Lua scripts to stock/operations.py. Keep all existing SAGA functions unchanged.

    1. PREPARE_STOCK_LUA:
       - KEYS[1] = item key {item:<item_id>}, KEYS[2] = hold key {item:<item_id>}:hold:<order_id>
       - ARGV[1] = quantity (string), ARGV[2] = new item bytes (pre-computed), ARGV[3] = expected current raw bytes (CAS)
       - Idempotency: if hold key exists -> return "ALREADY_PREPARED"
       - Check item exists, CAS compare raw bytes, atomic deduct + SET hold key with 604800 TTL (7 days)
       - Returns: "OK", "ALREADY_PREPARED", "RETRY", or JSON error string

    2. COMMIT_STOCK_LUA:
       - KEYS[1] = hold key {item:<item_id>}:hold:<order_id>
       - Simply DEL hold key, return "OK" (idempotent -- succeeds even if already deleted)

    3. ABORT_STOCK_LUA:
       - KEYS[1] = item key, KEYS[2] = hold key
       - ARGV[1] = new item bytes (stock restored), ARGV[2] = expected current raw bytes (CAS)
       - If hold key gone -> return "ALREADY_ABORTED"
       - Read hold quantity, CAS check item bytes, restore stock + DEL hold key
       - Returns: "OK", "ALREADY_ABORTED", "RETRY"

    4. prepare_stock(db, item_id: str, quantity: int, order_id: str) -> dict:
       - item_key = f"{{item:{item_id}}}"
       - hold_key = f"{{item:{item_id}}}:hold:{order_id}"
       - CAS retry loop (same pattern as reserve_stock):
         - Read current item bytes, decode StockValue, check stock >= quantity
         - If insufficient: return {"success": False, "error_message": "insufficient stock"}
         - Compute new StockValue with decremented stock, encode
         - eval PREPARE_STOCK_LUA with 2 keys, 3 args
         - Handle OK/ALREADY_PREPARED/RETRY/error responses
       - Return {"success": True/False, "error_message": ""}

    5. commit_stock(db, item_id: str, order_id: str) -> dict:
       - hold_key = f"{{item:{item_id}}}:hold:{order_id}"
       - eval COMMIT_STOCK_LUA with 1 key
       - Return {"success": True, "error_message": ""}

    6. abort_stock(db, item_id: str, order_id: str) -> dict:
       - CAS retry loop:
         - Check hold key exists (GET). If None -> return success (already aborted)
         - Read hold quantity (int), read current item bytes, decode, add back quantity, encode
         - eval ABORT_STOCK_LUA with 2 keys, 2 args
         - Handle OK/ALREADY_ABORTED/RETRY
       - Return {"success": True/False, "error_message": ""}
