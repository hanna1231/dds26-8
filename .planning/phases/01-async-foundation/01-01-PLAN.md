  <name>Task 1: Migrate Order service app.py to Quart+async</name>
  <files>order/app.py, order/requirements.txt</files>
  <action>
Migrate order/app.py from Flask to Quart with async Redis and httpx. Apply these specific changes:

**Import swap:**
- `from flask import Flask, jsonify, abort, Response` -> `from quart import Quart, jsonify, abort, Response`
- `import redis` -> `import redis.asyncio as redis`
- `import requests` -> `import httpx`
- Remove `import atexit`
- `app = Flask("order-service")` -> `app = Quart("order-service")`

**Redis client lifecycle (replace module-level db and atexit):**
- Remove the module-level `db: redis.Redis = redis.Redis(...)` initialization
- Remove the `close_db_connection()` function and `atexit.register(close_db_connection)`
- Declare `db: redis.Redis = None` as module-level placeholder
- Declare `http_client: httpx.AsyncClient = None` as module-level placeholder
- Add `@app.before_serving` async startup hook that does:
  ```python
  @app.before_serving
  async def startup():
      global db, http_client
      db = redis.Redis(host=os.environ['REDIS_HOST'],
                       port=int(os.environ['REDIS_PORT']),
                       password=os.environ['REDIS_PASSWORD'],
                       db=int(os.environ['REDIS_DB']))
      http_client = httpx.AsyncClient()
  ```
- Add `@app.after_serving` async shutdown hook:
  ```python
  @app.after_serving
  async def shutdown():
      await db.aclose()
      await http_client.aclose()
  ```

**Make all route handlers and helpers async:**
- Every `def` that calls `db.*` or `http_client.*` becomes `async def`
- Every `db.get(...)`, `db.set(...)`, `db.mset(...)` call gets `await` prepended
- `get_order_from_db` -> `async def get_order_from_db` with `await db.get(order_id)`
- `send_post_request` -> `async def send_post_request` with `await http_client.post(url)` and `except httpx.RequestError` (not `requests.exceptions.RequestException`)
- `send_get_request` -> `async def send_get_request` with `await http_client.get(url)` and `except httpx.RequestError`
- `rollback_stock` -> `async def rollback_stock` with `await send_post_request(...)` in the loop
- All route handlers: `async def create_order`, `async def batch_init_users`, `async def find_order`, `async def add_item`, `async def checkout`
- All callers of helpers must `await` them: `await get_order_from_db(...)`, `await send_post_request(...)`, etc.

**httpx response compatibility:**
- `item_reply.status_code` works identically in httpx (no change needed)
- `item_reply.json()` works identically in httpx (no change needed)
- `stock_reply.status_code` works identically (no change needed)
- `user_reply.status_code` works identically (no change needed)

**Preserve abort() calls exactly as-is.** `abort()` does NOT need `await` -- it raises HTTPException synchronously in both Flask and Quart. Keep `return abort(400, DB_ERROR_STR)` patterns unchanged.

**Update __main__ and remove gunicorn logger block:**
- Replace `if __name__ == '__main__': app.run(host="0.0.0.0", port=8000, debug=True)` with:
  ```python
  if __name__ == '__main__':
      app.run(host="0.0.0.0", port=5000, debug=True)
  ```
- Remove the entire `else:` block (gunicorn_logger wiring). Quart's app.logger works without wiring under Uvicorn.

**Do NOT change:** OrderValue Struct, msgspec imports, msgpack encode/decode calls, route paths, response formats, error strings, any business logic.

**Update order/requirements.txt** to:
```
quart==0.20.1
uvicorn==0.34.0
redis[hiredis]==5.0.3
msgspec==0.18.6
httpx==0.27.0
```
Remove Flask, gunicorn, requests.

- Log level `info` (no `=` sign -- uvicorn uses space-separated flag)
- No `--timeout` equivalent needed (uvicorn handles this differently)
- No `--reload` flag (production mode, and uvicorn disallows --reload with --workers)
