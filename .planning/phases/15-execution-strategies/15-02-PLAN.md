    Imports: `import asyncio`, `import logging`, `from workflow_types import WorkflowDefinition, WorkflowStep`, `from workflow_store import WorkflowStore`

    Note: TwoPhaseStrategy does NOT import retry.py. 2PC prepare is fire-once concurrent (no bounded retry on prepare); phase-2 commit/abort uses gather with return_exceptions which handles failures differently than SAGA's sequential retry.

    Class: `TwoPhaseStrategy` (stateless, no constructor params):

    Method `_validate_transition(self, from_state: str, to_state: str) -> None`:
    - Check `from_state in TPC_VALID_TRANSITIONS and to_state in TPC_VALID_TRANSITIONS[from_state]`
    - Raise `ValueError(f"Invalid 2PC transition: {from_state} -> {to_state}")` if invalid (per D-06)

    Method `async def execute(self, workflow_id: str, definition: WorkflowDefinition, context: dict, store: WorkflowStore) -> dict` (per D-01, D-03):

    Phase 1 - Prepare (concurrent):
    - Validate and transition INIT -> PREPARING: `self._validate_transition("INIT", "PREPARING")`, `await store.transition(workflow_id, "INIT", "PREPARING")`
    - Build prepare futures by calling each step's action: `futures = [step.action(context) for step in definition.steps]` (per Pattern 4 note in RESEARCH: calling at comprehension time captures step correctly)
    - `results = await asyncio.gather(*futures, return_exceptions=True)`
    - Evaluate results (per grpc_server.py:411-424 pattern):
      ```python
      all_yes = True
      first_error = ""
      for i, r in enumerate(results):
          if isinstance(r, Exception):
              all_yes = False
              if not first_error:
                  first_error = str(r)
          elif not r.get("success"):
              all_yes = False
              if not first_error:
                  first_error = r.get("error_message", "prepare failed")
          else:
              await store.mark_step_done(workflow_id, i)
      ```

    Phase 2a - Commit (if all_yes):
    - WAL: validate and transition PREPARING -> COMMITTING (WAL decision write BEFORE phase-2 messages per Pattern 3)
    - Send commit: `commit_futures = [step.action(context) for step in definition.steps]` then `await asyncio.gather(*commit_futures, return_exceptions=True)`
    - Transition COMMITTING -> COMMITTED
    - Return `{"success": True, "error_message": ""}`

    Phase 2b - Abort (if not all_yes):
    - WAL: validate and transition PREPARING -> ABORTING (WAL decision write BEFORE phase-2 messages)
    - Send abort: `abort_futures = [step.compensation(context) for step in definition.steps]` then `await asyncio.gather(*abort_futures, return_exceptions=True)`
    - Transition ABORTING -> ABORTED
    - Return `{"success": False, "error_message": first_error}`

    Per D-03: TwoPhaseStrategy does NOT have a separate compensate method. Abort is integral to execute.
