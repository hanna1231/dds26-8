    Class: `SagaStrategy` (stateless, no constructor params):

    Method `_validate_transition(self, from_state: str, to_state: str) -> None`:
    - Check `from_state in VALID_TRANSITIONS and to_state in VALID_TRANSITIONS[from_state]`
    - Raise `ValueError(f"Invalid SAGA transition: {from_state} -> {to_state}")` if invalid (per D-06)

    Method `async def execute(self, workflow_id: str, definition: WorkflowDefinition, context: dict, store: WorkflowStore) -> dict` (per D-01):
    - Maintain `completed_step_indices: list[int] = []`
    - For each `(i, step)` in `enumerate(definition.steps)`:
      - Call `result = await retry_forward(lambda s=step, c=context: s.action(c))` (lambda default-arg capture per Pattern 4)
      - If `not result.get("success")`:
        - Validate and transition to COMPENSATING: `self._validate_transition(current_state, "COMPENSATING")` then `await store.transition(workflow_id, current_state, "COMPENSATING")`
        - Call `await self.compensate(workflow_id, definition, context, store, completed_indices=completed_step_indices)`
        - Return `{"success": False, "error_message": result.get("error_message", "")}`
      - On success: call `await store.mark_step_done(workflow_id, i)`, append `i` to `completed_step_indices`
      - Determine intermediate state name: for a generic approach, use the step index to pick from a state progression. The state transitions for SAGA are domain-specific (STARTED -> STOCK_RESERVED -> PAYMENT_CHARGED -> COMPLETED). Since the strategy must generalize to any WorkflowDefinition, compute intermediate states as: after step `i` succeeds, transition from current state to the next state in definition order. Store the state progression as a list derived from VALID_TRANSITIONS keys: `["STARTED", "STOCK_RESERVED", "PAYMENT_CHARGED", "COMPLETED"]`. After step `i`, transition from `states[i]` to `states[i+1]`.
      - Note: The state list maps 1:1 with steps: step 0 transitions STARTED->STOCK_RESERVED, step 1 transitions STOCK_RESERVED->PAYMENT_CHARGED, step N-1 transitions to COMPLETED. Build this list as `STATE_SEQUENCE = ["STARTED", "STOCK_RESERVED", "PAYMENT_CHARGED", "COMPLETED"]` at module level. `execute()` starts with assumption that current state is `STATE_SEQUENCE[0]` and after each step `i` transitions from `STATE_SEQUENCE[i]` to `STATE_SEQUENCE[i+1]`.
    - After all steps succeed: validate and transition to COMPLETED, return `{"success": True, "error_message": ""}`

    Method `async def compensate(self, workflow_id: str, definition: WorkflowDefinition, context: dict, store: WorkflowStore, *, completed_indices: list[int] | None = None) -> dict` (per D-02):
    - If `completed_indices is None` (recovery path per Pitfall 2): call `current = await store.get(workflow_id)`, build indices from `step_N_done` flags: `completed_indices = [i for i in range(len(definition.steps)) if current.get(f"step_{i}_done") == "1"]`
    - Iterate `reversed(completed_indices)`:
      - `step = definition.steps[i]`
      - Call `await retry_forever(lambda s=step, c=context: s.compensation(c))` (infinite retry per STR-02, lambda default-arg capture)
    - Validate and transition from COMPENSATING to FAILED
    - Return `{"success": False, "error_message": "compensated"}`
