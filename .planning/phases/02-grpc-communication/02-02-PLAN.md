**StockValue struct:** Import from app.py or redefine. Since app.py defines `StockValue`, import it: `from app import StockValue`. If circular import is an issue, redefine the struct in grpc_server.py.

**Idempotency Lua script** (per locked decision: single atomic check-and-set, no TOCTOU):
```python
IDEMPOTENCY_ACQUIRE_LUA = """
local existing = redis.call('GET', KEYS[1])
if existing then
    return existing
end
redis.call('SET', KEYS[1], '__PROCESSING__', 'EX', ARGV[1])
return '__NEW__'
"""
```
- KEYS[1]: `idempotency:{idempotency_key}`
- ARGV[1]: short TTL for processing lock (30 seconds)
- Returns cached JSON result if key exists, `__NEW__` if this is the first call
- After business logic completes, SET the key with the real result JSON and 24h TTL

**StockServiceServicer class:**
- `__init__(self, db)`: store redis connection
- `async def ReserveStock(self, request, context)`:
  1. Build ikey = `idempotency:{request.idempotency_key}`
  2. Run IDEMPOTENCY_ACQUIRE_LUA with ikey, ARGV=[30]
  3. If result is not `__NEW__` and not `__PROCESSING__`: return cached result (json.loads → StockResponse)
  4. If `__PROCESSING__`: return StockResponse(success=False, error_message="operation in progress, retry")
  5. If `__NEW__`: execute business logic:
     - GET item_id from Redis, decode with msgpack (StockValue)
     - If not found: store fail result, return StockResponse(success=False, error_message="item not found")
     - If stock < quantity: store fail result, return StockResponse(success=False, error_message="insufficient stock")
     - Subtract stock, SET back to Redis with msgpack encode
     - Store success result: `SET ikey json.dumps({"success": True, "error_message": ""}) EX 86400`
     - Return StockResponse(success=True, error_message="")
- `async def ReleaseStock(self, request, context)`: Same idempotency pattern, but ADD stock instead of subtract
- `async def CheckStock(self, request, context)`: No idempotency (read-only). GET item_id, decode, return CheckStockResponse with stock and price

**IMPORTANT business logic details (match existing app.py behavior):**
- Stock values are stored as msgpack-encoded StockValue structs keyed by item_id
- `item_entry.stock -= quantity` then check `< 0` (match remove_stock pattern in app.py)
- For ReleaseStock: `item_entry.stock += quantity` (match add_stock pattern)
- Use `msgpack.decode(entry, type=StockValue)` and `msgpack.encode(item_entry)` exactly as in app.py

**Error handling per locked decision:** Business errors in response fields (success=False + error_message), NOT gRPC status codes. Never call `context.abort()` for business failures.

**Module-level server variable + serve/stop functions:**
```python
_grpc_server: grpc.aio.Server = None

async def serve_grpc(db) -> None:
    global _grpc_server
    _grpc_server = grpc.aio.server()
    add_StockServiceServicer_to_server(StockServiceServicer(db), _grpc_server)
    _grpc_server.add_insecure_port("[::]:50051")
    await _grpc_server.start()
    await _grpc_server.wait_for_termination()

async def stop_grpc_server():
    if _grpc_server is not None:
        await _grpc_server.stop(grace=5.0)
```

**PaymentServiceServicer class:**

- `async def ChargePayment(self, request, context)`:

  2. GET user_id from Redis, decode with msgpack (UserValue)
